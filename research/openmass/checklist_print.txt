CAPTURE CHECKLIST — builds needed to break the totalVolume collinearity
Derived from prun-openmass/ridge_output.txt (rank 45/60, 15 redundant columns).
Goal: after capturing these blueprints and re-pooling, the component-count matrix
becomes full-rank enough that a fit can DERIVE totalVolume from a loadout — which is
the prerequisite for computing hypothetical-mode empty mass without a manual field.
================================================================================

HOW TO READ THIS
  * Each build is a ONE-CHANGE delta from a fixed REFERENCE ship (below) unless it
    says otherwise. Change exactly what is listed; leave everything else identical.
  * Quantities are the functional modules YOU select in the blueprint designer.
  * Hull PLATES and SSC (structuralSpacecraftComponent) are AUTO-derived by the game
    from the resulting totalVolume — you do NOT set them. You only pick the plate TYPE
    (basic/reinforced/lightweight/advanced/hardened). They still get captured.
  * After building each, open it so BLUEPRINT_BLUEPRINTS is captured (performance +
    full bill of material), same as the existing 87.

WHY ONE-CHANGE DELTAS: a component's coefficient is unidentifiable when its column is
an exact linear combination of other columns across all 87 builds. Adding one build
where that component takes a value the old relation does NOT predict makes its column
linearly independent — so its own effect on totalVolume becomes measurable.

ROOT-CAUSE NOTE — decompose BP-CLNY-0000
BP-CLNY-0000 is the ONLY blueprint containing HAM, VFT, VOE (and the sole carrier of
several other exotics + full shield stacks). Because those parts occur together in
exactly one ship, their coefficients are pinned by a single point — this is why
cross-validation failed worst (-84%) on CLNY. Section A rebuilds each of those exotic
parts into a SEPARATE ordinary ship so it stops being a one-point mystery.
================================================================================

SECTION B — SLOT-SUM IDENTITIES (common parts made redundant by rigid ship structure)
Every one of the 87 has exactly one bridge and exactly one of each slot, so e.g.
"STL tank sum = bridge sum" holds identically and hides one option per slot. To break
these you must (i) swap the slot type at fixed size, (ii) where the designer allows,
put 0 or 2 of a slot, and (iii) capture a pure-STL ship (no FTL parts at all).
────────────────────────────────────────────────────────────────────────────────

    remove FFC, SFE, MFE, LFE, STS, RCT (reactor) and SFL (FTL tank).

    "FTL-part count == bridge count" relation at once. Capture 2-3 of these at

    class here.

B2  Hull RHP.  REFERENCE, except STL tank MSL -> SSL (smallStlTank x1).

SECTION C — HULL-TIER COVERAGE + SSC/PLATE DECOUPLING
Targets: HULL_ADVANCED, HULL_HARDENED, HULL_REINFORCED redundancy, and the {SSC, plate}
group. ADVANCED (AHP) and HARDENED (HHP) each have only 2 blueprints today; a single
point cannot separate a hull-tier's effect from confounds (this is exactly why CLNY,
the lone big ADVANCED ship, drove the worst CV error).
NOTE: you cannot set plate/SSC counts directly — vary hull TYPE and overall SIZE. Across
a size range the plate-count rule (~0.535*V^0.655) and the SSC rule (~0.0478*V) diverge,
which is what separates the SSC column from the plate column.
────────────────────────────────────────────────────────────────────────────────

C1-C5  Five ADVANCED-hull (AHP) ships spanning small->large totalVolume. Use the
       REFERENCE functional set but change cargo to sweep size:
         C1  AHP hull, cargo SCB x1   (small)
         C2  AHP hull, cargo LCB x1   (medium — like the reference)
         C3  AHP hull, cargo HCB x1   (large)
         C4  AHP hull, cargo LCB x2   (or MCB x2) — different size point
         C5  AHP hull, cargo SCB x1 + no FTL parts (pure-STL advanced) — extra spread
       Targets: HULL_ADVANCED (currently pinned by 1 real ship). Five spread points let
       the advanced-plate coefficient be estimated instead of absorbing one ship's noise.

C6-C10 Five HARDENED-hull (HHP) ships spanning small->large, same size sweep as C1-C5
       (SCB, LCB, HCB, LCB x2, and one pure-STL). Targets: HULL_HARDENED (only 2 today).

C11    (Optional) 2-3 extra LIGHTWEIGHT (LHP) ships at sizes not yet sampled, and a
       couple more REINFORCED (RHP) at small sizes, to widen the plate-type x size grid
       and further separate SSC from the plate columns.

CAVEAT: breaking the collinearity is NECESSARY but may not be SUFFICIENT. totalVolume is
game-computed (a hull-envelope quantity) and the plates/SSC that dominate it are derived
FROM it, so the relationship of totalVolume to the functional loadout could be non-linear.
These captures make the linear fit IDENTIFIABLE; whether a linear model then predicts
held-out totalVolume within the 2%/5% target must be re-tested with 10-fold CV after
capture. If it still fails, the next step is a non-linear/functional model, not more data.
