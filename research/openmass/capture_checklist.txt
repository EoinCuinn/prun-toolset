WHY THIS IS NOW SHORT
  The selections-only re-analysis (see selections_audit.txt) showed the earlier
  "collinearity" was largely an ARTEFACT of the old feature matrix, which included
  game-derived hull-plate/SSC counts that scale with totalVolume. Rebuilt from the
  user-controlled dropdowns, the model already predicts totalVolume within 2% for
  76 of 87 blueprints. The only real gap is UNDER-SAMPLED dropdown options: 18-19
  options appear in <=3 blueprints today, so when one is held out in cross-validation
  it has too little support and that prediction breaks.
  Fix = simply build a few more of each. No pure-STL ships, no slot-swap
  "collinearity-breaking" builds (v1 Section B) are needed.

  Goal: bring every under-sampled option to >=5 blueprints total. That takes the 15
  builds below. Each build is a normal ship; the table just tells you what to pick in
  each dropdown of the blueprint designer. After building + opening all 15 (so they
  get captured), re-pool and re-run the selections-only 10-fold CV.

CAVEATS / NOTES
  * Habitation module is a large "colony" part. If the designer won't place it on a
    small ship (rows 2, 11, 14 use small cargo), raise that row's Cargo to Large or
    Huge — the aim is to capture the habitat module; the cargo size is incidental.
  * Sizes: because Cargo varies row-to-row, the two thin hull tiers (Advanced,
    Hardened) get sampled at several sizes rather than one point — that is the fix for
    the BP-CLNY-0000 lone-advanced-ship problem, so keep the cargo varied as shown.
  * If the designer rejects any specific combination, adjust that row and note which
    option still needs samples — every row's job is only to add one instance of its
    rare picks; the common slots can be anything valid.
  * After capturing all 15: re-pool to 102 blueprints and re-run the selections-only
    ridge + 10-fold CV (the Step 2/3 method in selections_audit.txt). Expect it to pass
    the 2% / 5% threshold once every option a build uses has >=5 examples. If a residual
    non-linearity remains (watch the one common-option outlier BP-VUNR-1627), the next
    step is a non-linear model, not more captures.
